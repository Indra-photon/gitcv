# Resume Formatting Guide

## Using Markdown for Text Formatting

Your resume editor now supports **rich text formatting** using simple markdown syntax. You can bold, italicize, and emphasize text directly in the editor.

---

## Supported Formatting

### **Bold Text**
Make text bold to emphasize important keywords, technologies, or achievements.

**How to use:**
- Type `**text**` or `__text__`
- Example: `Built a **scalable** microservices architecture`
- Result: Built a **scalable** microservices architecture

**When to use bold:**
- ‚úÖ Important technologies: `**React**, **Node.js**, **AWS**`
- ‚úÖ Key metrics: `Increased performance by **40%**`
- ‚úÖ Critical achievements: `**Led a team of 5 developers**`
- ‚úÖ Action verbs: `**Developed**, **Implemented**, **Optimized**`

---

### **Italic Text**
Use italics for subtle emphasis or to highlight terms.

**How to use:**
- Type `*text*` or `_text_`
- Example: `Implemented *continuous integration* pipeline`
- Result: Implemented *continuous integration* pipeline

**When to use italic:**
- ‚úÖ Technical terms: `Used *microservices architecture*`
- ‚úÖ Soft skills: `Demonstrated *leadership* and *communication*`

---

### **Combining Bold and Italic**
You can combine both for extra emphasis.

**How to use:**
- Type `***text***` or `**_text_**`
- Example: `***Critical feature*** delivered on time`
- Result: ***Critical feature*** delivered on time

---

## Best Practices

### ‚úÖ **DO:**
- **Highlight key achievements:** `Reduced load time by **50%**`
- **Emphasize important tech:** `Built with **React**, **TypeScript**, and **GraphQL**`
- **Make metrics stand out:** `Managed **$2M budget** and **10+ team members**`
- **Bold action verbs:** `**Architected** and **deployed** cloud infrastructure`

### ‚ùå **DON'T:**
- **Over-format:** Don't make every word bold - it loses impact
- **Mix too many styles:** Keep it professional and clean
- **Format entire sentences:** Only highlight key parts
